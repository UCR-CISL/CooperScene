Overview

CooperScene is the first real-world, multi-agent, multi-modal cooperative autonomy dataset with C-V2X communication characterization. It focuses on complex urban intersection scenarios involving three connected autonomous vehicles and one instrumented infrastructure roadside unit, all equipped with multi-modal sensors and commercial off-the-shelf C-V2X communication radios.

The dataset provides tightly aligned LiDAR, camera, GNSS/IMU, and C-V2X network data, enabling the community to jointly evaluate perception performance and the practical constraints imposed by real-world vehicular networks. CooperScene supports 3D object detection, tracking, and motion prediction benchmarks with synchronized pair-wise C-V2X throughput measurements.

59K Synchronized LiDAR Frames
18K Image Frames
344K 3D Bounding Box Labels
4 Cooperative Agents

Key Features

Multi-Agent Cooperative Scenes

High-quality interactive scenes from three vehicles and one infrastructure agent across diverse real-world urban intersection scenarios.

C-V2X Communication Traces

Real-world C-V2X network traces including latency, throughput, packet loss rate and jitter — enabling cross-disciplinary research.

Multi-Modal Sensing

LiDAR, camera, GNSS/IMU data streams per agent, well-calibrated and temporally synchronized with sub-millisecond accuracy.

Centimeter-Level Localization

GNSS-RTK positioning with spatial-temporal ICP alignment achieving an average RMSE of 0.2m across all agents.

Precise Synchronization

PTP-based time synchronization and hardware triggering ensure globally consistent timestamps across all platforms and sensors.

Unified Benchmark

Comprehensive benchmarks for detection, tracking, and motion prediction evaluating both perception accuracy and communication efficiency.

Dataset Format

CooperScene is released in two formats: our native format with global ENU labels and per-agent metadata, and an OPV2V-compatible format for direct use with cooperative perception frameworks such as OpenCOOD.

  • CooperScene Native Format
  • OPV2V Format
CooperScene/
  {scene_id}/ # e.g., 1, 2, ..., 14
    {agent_id}/ # 0, 1, 2, 3
      sensor/
        lidar/
          {frame_id}.pcd # LiDAR point cloud
      bbox/
        {frame_id}.json # Per-agent 3D bounding boxes
      labels/
        {frame_id}.yaml # Per-agent labels (lidar & ENU coords)
      metadata.json # Per-frame pose, GNSS, C-V2X throughput
      pairwise_icp.json # Pairwise ICP alignment results
      raw_gnss.csv # Raw GNSS trajectory
    labels_global/ # Global labels in ENU coordinates
      {frame_id}.yaml # Merged multi-agent annotations with track IDs

Key Fields

Global Labels

Each frame has globally merged 3D bounding boxes in ENU coordinates with quaternion orientation, dimensions (l, w, h), object class, track ID, speed, and ego-agent visibility list.

Agent Metadata

Per-frame quaternion pose, GNSS (lat/lon/alt), ENU position, and pair-wise C-V2X receive bytes and throughput for each agent link.

For compatibility with OpenCOOD and other cooperative perception frameworks, we also provide the dataset in OPV2V format. Each timestamp directory contains per-agent YAML files with pose and label information.

CooperScene_OPV2V/
  train/ # Training split
  validate/
  test/
    {scene_id}/
      {timestamp}/ # Globally synchronized frame
        {agent_id}.pcd # LiDAR point cloud
        {agent_id}.yaml # Pose, calibration, labels
        {agent_id}_camera0.png # Camera image

Benchmark

We benchmark state-of-the-art cooperative perception methods under five cooperative settings with varying numbers of agents. All models are retrained on CooperScene using the OpenCOOD framework.

Agent Configurations

V+V+I

V+2V
